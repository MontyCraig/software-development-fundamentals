# Unit Testing

## Overview
Unit testing involves testing individual components or modules of code in isolation.

## Components

### 1. Frameworks
- PyTest
  - Test discovery
  - Fixtures
  - Parameterization
  - Markers
- UnitTest
  - Test cases
  - Test suites
  - Assertions
  - Test runners
- Mock Frameworks
  - Mock objects
  - Stubs
  - Fakes
  - Spies

### 2. Practices
- Test-Driven Development (TDD)
  - Red-Green-Refactor cycle
  - Test first approach
  - Continuous testing
- Test Organization
  - Arrange-Act-Assert
  - Given-When-Then
  - Test naming conventions
- Code Coverage
  - Statement coverage
  - Branch coverage
  - Path coverage
  - Mutation testing

### 3. Patterns
- Test Fixture Patterns
  - Setup/Teardown
  - Object Mother
  - Test Data Builder
- Test Double Patterns
  - Dummy objects
  - Fake objects
  - Stub objects
  - Mock objects
- Test Organization Patterns
  - Four-Phase Test
  - Test Class per Class
  - Test Class per Feature
